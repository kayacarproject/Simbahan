7. TECHNICAL REQUIREMENTS
================================================================================

FRONTEND
  - Next.js 14 (App Router) — SSR for SEO-critical pages (home, events, readings)
  - TailwindCSS — reuse same design tokens
  - Zustand — port existing stores (authStore, bookmarkStore, churchStore)
  - React Query (TanStack) — server state, caching, background refetch
  - Framer Motion — page transitions, card animations

BACKEND
  - Next.js API Routes for lightweight endpoints
  - Supabase — PostgreSQL database + Auth + Storage + Realtime
  - Firebase Cloud Messaging — push notifications (web push)
  - PayMongo — Philippine payment gateway (GCash, PayMaya, cards)

DATABASE SCHEMA (Supabase)
  users           — id, email, name, role, avatar_url, parish_id, created_at
  families        — id, name, address, members[]
  announcements   — id, title, body, category, image_url, published_at
  events          — id, title, description, date, venue, category, poster_url
  mass_schedule   — id, day_of_week, time, type, celebrant, is_special
  donations       — id, user_id, cause_id, amount, method, status, created_at
  donation_causes — id, title, description, target_amount, image_url
  sacrament_reqs  — id, user_id, type, status, notes, scheduled_at
  novenas         — id, title, days[], audio_url
  notifications   — id, user_id, title, body, type, read, created_at
  media           — id, type (video/audio/photo), title, url, album_id

API STRUCTURE
  /api/auth/*              — login, register, reset
  /api/announcements/*     — CRUD
  /api/events/*            — CRUD + RSVP
  /api/mass-schedule/*     — read + admin update
  /api/donations/*         — create, history, causes
  /api/sacraments/*        — request, status update
  /api/novenas/*           — read
  /api/readings/today      — fetch from API (Universalis or custom)
  /api/notifications/*     — list, mark-read, send (admin)
  /api/media/*             — upload, list, delete
  /api/admin/*             — dashboard stats, user management

AUTHENTICATION
  - Supabase Auth (email/password + magic link)
  - JWT stored in httpOnly cookies
  - Row-Level Security (RLS) on all Supabase tables
  - Role-based access: guest | member | staff | admin

HOSTING
  - Vercel — frontend + API routes (zero-config Next.js)
  - Supabase Cloud — database + storage
  - Cloudflare — CDN + DDoS protection for media assets
